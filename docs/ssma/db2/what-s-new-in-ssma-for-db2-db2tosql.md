# <a name="what39s-new-in-ssma--for-db2-db2tosql"></a>Quel &#39; s de SSMA pour DB2 (DB2ToSQL)
Cette rubrique répertorie SSMA pour DB2 les modifications dans chaque version.  

## <a name="may-2016"></a>Mai 2016  
La version de mai 2016 de SSMA pour DB2 contient les modifications suivantes :  

1.  Prise en charge pour SQL Server 2016
2.  Conversion ajoutée des tables en mémoire et régulières de DB2 aux fonctionnalités en mémoire et hekaton de SQL Server
3.  Ajout de conversion DB2 des contrôles d’accès aux objets de stratégie de SQL Server (sécurité de niveau ligne pour DB2)
4.  Ajout de conversion de tables avec version système DB2 aux tables temporelles de SQL Server
5.  Programme de résolution et l’Analyseur de DB2 amélioré
6.  Supprimer la vérification du programme d’installation pour .net 2.0
7.  Supprimé *.dll inutiles à partir du programme d’installation de Db2
8.  Fixe « enregistrer le projet » et « ouvrir le projet » des commandes pour la Console de SSMA
9.  Commande securepassword « fixe » pour la Console de SSMA
10. Fixe le comptage des objets pour le chargement initial
11. Résolution du bogue dans les paramètres globaux
  
## <a name="march-2016"></a>Mars 2016  
La version préliminaire de mars 2016 de SSMA pour DB2 contient les modifications suivantes :  
  
1.  Prend en charge la migration vers SQL Server 2016  
  
## <a name="january-2016"></a>Janvier 2016  
La version de maintenance de janvier 2016 de SSMA pour DB2 contient les modifications suivantes :  
  
1.  Prise en charge un nombre de fonctions standards  
  
2.  Erreurs de l’analyseur DB2 fixe  
  
3.  ZOS de v9 fixe DB2 prise en charge (RFC 5690920)  
  
4.  DB2 fixe non résolu les erreurs identificateur lors de la conversion  
  
5.  Élément de Menu journal View ajouté SSMA (RFC 5706203)  
  
6.  Ajout de télémétrie  
  
## <a name="november-2014"></a>Novembre 2014  
La version de novembre 2014 de SSMA pour DB2 est le premier de ce type.
